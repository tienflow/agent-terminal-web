# Agent Terminal 离线部署指南

本文档用于把当前 Web 终端服务部署到其他服务器，前提是：
- 目标服务器没有外网
- 目标服务器已经安装 Docker
- 你可以从一台有网机器把文件传到目标服务器

当前服务特性：
- 单容器部署
- 浏览器 Web 终端
- `POST /exec` 可供 Agent 调用
- 实际执行的是宿主机 shell，不是容器内 shell

## 目录结构

当前工件目录：

```text
agent-terminal/
├── app.py
├── index.html
├── Dockerfile
└── requirements.txt
```

## 总体思路

离线部署不要在目标服务器现拉基础镜像或 Python 依赖。
正确做法是：

1. 在有网机器上构建 Docker 镜像
2. 导出为 tar 包
3. 把 tar 包传到无网服务器
4. 在无网服务器 `docker load`
5. 用固定参数启动容器

## 一. 在有网机器构建镜像

进入项目目录：

```bash
cd /path/to/agent-terminal
```

构建镜像：

```bash
docker build -t agent-terminal:latest .
```

建议顺手确认镜像存在：

```bash
docker images | grep agent-terminal
```

## 二. 导出镜像为离线包

导出：

```bash
docker save -o agent-terminal-latest.tar agent-terminal:latest
```

可选：压缩后再传，减小体积。

```bash
gzip -f agent-terminal-latest.tar
```

得到文件之一：

- `agent-terminal-latest.tar`
- 或 `agent-terminal-latest.tar.gz`

## 三. 传到无网服务器

用你现有的传输方式把文件传过去，例如：

```bash
scp agent-terminal-latest.tar hw2:/root/
```

如果是压缩包：

```bash
scp agent-terminal-latest.tar.gz hw2:/root/
```

## 四. 在无网服务器导入镜像

登录目标服务器后执行。

如果传的是 tar：

```bash
docker load -i /root/agent-terminal-latest.tar
```

如果传的是 tar.gz：

```bash
gunzip -f /root/agent-terminal-latest.tar.gz
docker load -i /root/agent-terminal-latest.tar
```

确认镜像已经导入：

```bash
docker images | grep agent-terminal
```

## 五. 启动容器

先删掉旧容器，避免名称冲突：

```bash
docker rm -f agent-terminal 2>/dev/null || true
```

按当前线上一致参数启动：

```bash
docker run -d \
  --name agent-terminal \
  --restart unless-stopped \
  --privileged \
  --pid=host \
  -p 7681:7681 \
  agent-terminal:latest
```

这些参数的含义：
- `--privileged`：允许容器访问宿主机命名空间
- `--pid=host`：让 `nsenter -t 1` 进入宿主机 1 号进程命名空间
- `-p 7681:7681`：暴露 Web 终端端口
- `--restart unless-stopped`：机器重启后自动拉起

## 六. 验证部署

### 1. 检查容器状态

```bash
docker ps | grep agent-terminal
```

### 2. 检查健康接口

本机验证：

```bash
curl -s http://127.0.0.1:7681/health
```

预期类似：

```json
{"status":"ok","pty":true,"time":"11:30:00"}
```

### 3. 检查 Agent 调用

```bash
curl -s -X POST http://127.0.0.1:7681/exec \
  -H 'Content-Type: application/json' \
  -d '{"cmd":"hostname && whoami"}'
```

### 4. 浏览器验证

浏览器打开：

```text
http://<服务器IP>:7681
```

你应该能看到：
- Web 终端页面
- 右上角 `connected`
- 宿主机 shell 输出

## 七. 更新版本

如果你后续改了 `app.py` 或 `index.html`，离线更新流程也是一样：

1. 有网机器重新 `docker build`
2. 重新 `docker save`
3. 传到目标服务器
4. `docker load`
5. 重建容器

示例：

```bash
docker rm -f agent-terminal
docker load -i /root/agent-terminal-latest.tar
docker run -d \
  --name agent-terminal \
  --restart unless-stopped \
  --privileged \
  --pid=host \
  -p 7681:7681 \
  agent-terminal:latest
```

## 八. 常见问题

### 1. 页面能打开，但没有连接

先看日志：

```bash
docker logs -f agent-terminal
```

重点看是否有：
- `PTY started`
- `Agent-Terminal started`
- `WebSocket /ws [accepted]`

### 2. /exec 能调，但不是宿主机环境

先确认启动参数是否完整，尤其是这两个：

```text
--privileged
--pid=host
```

少一个都可能导致 `nsenter` 行为不对。

### 3. 服务器没有外网，构建失败

不要在目标服务器构建。
必须在有网机器构建完，再把镜像 tar 包带过去。

### 4. 端口冲突

如果 `7681` 被占用，可以换宿主机端口，例如：

```bash
docker run -d \
  --name agent-terminal \
  --restart unless-stopped \
  --privileged \
  --pid=host \
  -p 9001:7681 \
  agent-terminal:latest
```

这时访问地址变成：

```text
http://<服务器IP>:9001
```

## 九. 推荐交付物

如果你要把这个服务交给别人离线部署，建议一起交付这 2 个东西：

1. `agent-terminal-latest.tar.gz`
2. 本文档

这样对方即使完全没外网，也能直接部署。
